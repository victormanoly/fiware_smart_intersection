Globale Beschreibung: **Diese Einheit enthält eine harmonisierte geografische und kontextuelle Beschreibung einer Straße.**  

## Liste der Eigenschaften  

- `address`: Die Postanschrift  - `alternateName`: Ein alternativer Name für diesen Artikel  - `annotations`: Anmerkungen zum Artikel  - `areaServed`: Das geografische Gebiet, in dem eine Dienstleistung oder ein angebotener Artikel erbracht wird  - `color`: Die Farbe des Produkts  - `dataProvider`: Eine Folge von Zeichen, die den Anbieter der harmonisierten Dateneinheit identifiziert.  - `dateCreated`: Zeitstempel der Entitätserstellung. Dieser wird normalerweise von der Speicherplattform zugewiesen.  - `dateModified`: Zeitstempel der letzten Änderung der Entität. Dieser wird in der Regel von der Speicherplattform vergeben.  - `description`: Eine Beschreibung dieses Artikels  - `id`: Eindeutiger Bezeichner der Entität  - `image`: Ein Bild des Artikels  - `length`: Gesamtlänge dieser Straße in Kilometern  - `location`: Geojson-Referenz auf das Element. Es kann Punkt, LineString, Polygon, MultiPoint, MultiLineString oder MultiPolygon sein  - `name`: Der Name dieses Elements.  - `owner`: Eine Liste mit einer JSON-kodierten Zeichenfolge, die auf die eindeutigen Ids der Eigentümer verweist  - `refRoadSegment`: Straßensegmente, die diese Straße definieren. Liste der Referenzen auf Entitäten des Typs RoadSegment  - `responsible`: Verantwortlich für die Straße, d. h. der Organismus oder das Unternehmen, das für die Instandhaltung der Straße verantwortlich ist  - `roadClass`: Die Klassifizierung dieser Straße. Enum:'Autobahn, primär, Anliegerstraße, sekundär, Anliegerstraße, tertiär, Fernstraße, unklassifiziert'.  Erlaubte Werte: Die von [OpenStreetMap](http://wiki.openstreetmap.org/wiki/Key:highway) beschriebenen.  - `seeAlso`: Liste von uri, die auf zusätzliche Ressourcen über das Element verweist  - `source`: Eine Folge von Zeichen, die die ursprüngliche Quelle der Entitätsdaten als URL angibt. Empfohlen wird der voll qualifizierte Domänenname des Quellanbieters oder die URL zum Quellobjekt.  - `type`: NGSI-Entitätstyp. Es muss Road sein    
Erforderliche Eigenschaften  
Straßen bestehen aus einer oder mehreren RoadSegment-Entitäten. Straßensegmente werden normalerweise verwendet, um z. B. die verschiedenen Fahrbahnen von Autobahnen zu modellieren. Das Vorhandensein dedizierter Fahrradwege sollte ebenfalls mit Straßensegmenten modelliert werden. Straßensegmente spielen auch eine wichtige Rolle bei der Modellierung von Straßen mit heterogenen Segmenten, z. B. Segmente, auf denen unterschiedliche Geschwindigkeitsbegrenzungen gelten. Diese Einheit ist in erster Linie mit den vertikalen Segmenten Automotive und Smart City und den damit verbundenen IoT-Anwendungen verbunden. Dieses Datenmodell wurde in Zusammenarbeit mit Mobilfunkbetreibern und der GSMA entwickelt.
